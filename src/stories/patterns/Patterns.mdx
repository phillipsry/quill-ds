# Patterns

Ready-made, **Quill-styled** compositions built entirely from the components and tokens —
so they inherit the theme and dark mode automatically, and pass the same enforced
accessibility checks. Use them as starting points and adapt.

## Catalog

- **Auth** — Login · Login variants (split panel, minimal) · Sign up · Signup, social first · Forgot password · OTP verification
- **Forms** — Settings · Wizard · Contact form · Newsletter signup · File upload · Checkout
- **Data** — Stat cards · Data table · Notifications · Order summary · Chat · Profile card · Search results · Kanban board · Analytics charts · Activity feed · Invoice · Calendar page · Calendar range
- **State** — Empty state · Error 404 · Cookie consent · Alerts · Onboarding checklist
- **Marketing** — Hero · Feature section · Pricing · Testimonial · Footer · FAQ · Team section · Stats band · Announcement banner
- **Shells** — Dashboard · Page header · Tabs page · List + detail · Sidebar navigation · Mail inbox
- **Nav** — Navbar · Command palette

## Install any pattern

Every pattern is published to the Quill registry as an installable block. Add one to
your project with the shadcn CLI — its source lands in `components/quill/`, and the
components it composes (plus the Quill `Icon`) install alongside it:

```bash
npx shadcn@latest add https://www.quilldesignsystem.com/r/checkout.json
```

Swap `checkout` for any block name — kebab-case of the pattern name, e.g.
`sidebar-nav`, `login-split-panel`, `analytics-charts`, `mail-shell`.

## How they’re built

Each pattern lives as real source in `registry/blocks/` and composes
`@/components/ui/*`; the stories here render that exact shipped code. Nothing is
hardcoded — colors, spacing, radii, and type resolve through the token foundation,
so a theme change propagates everywhere.
